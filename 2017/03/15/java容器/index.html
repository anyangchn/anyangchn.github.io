Java容器概述

发表于 2017-03-15 | 分类于 Java |

List

List是有序序列，允许插入重复元素。

ArrayList LinkedList Vector/Stack
动态数组实现的顺序存储，默认初始化长度为10，数组会自动扩容（1.5倍+1）；

可以通过ensureCapacity(int) 和 trimToSize()调整数组大小		 双向链表存储，get方法很慢 古老的java集合类，线程安全的，性能较差

排序

  • Collections.sort函数在1.8中改为了调用List的sort方法，之前List中不含sort方法。
  • sort方法先把集合转为数组，然后使用Arrays.sort进行排序，最后再更新集合。Arrays.sort在1.7版本前采用归并排序，1.7版本后改用timsort（适应性归并）
  • ArrayList重写了sort方法，直接对成员数组进行排序。

    回头看看timsort原理

区别

  • ArrayList和LinkedArrayList都是线程不安全的，Vector和Stack是线程安全的；
  • ArrayList在随机访问get()和set()上性能优越；
  • LinkedList在增删add()和remove()上性能优越。

Queue

Map

HashMap LinkedHashMap TreeMap WeekHashMap IdentityHashMap
哈希表的拉链法实现（数组+链表）

不支持有序性操作

支持null作为key和value，所以应该用containsKey()来判断是否存在该键		 继承至HashMap，使用双向链表维护了key-value对的插入顺序（Entry< k, v > before, after)

如果按顺序插入的话支持有序性操作		 红黑树结构

支持有序性操作		 key只保留对实际对象的弱引用，如果key所引用的对象没有被其他强引用变量所引用，可能会被垃圾回收，WeekHashMap也会自动删除这些key所对应的key-value值 对key相等的要求更严格，只有key1==key2时才认为两个key相等

普通HashMap只要求key1、key2非空时二者equals()。

HashMap与TreeMap、LinkedHashMap性能对比

  • TreeMap通常慢于HashMap，尤其是增删更慢，因为TreeMap要维护红黑树的次序。
  • 通常在需要有序性操作时才选择用TreeMap。
  • LinkedHashMap略慢于HashMap，但迭代遍历Map性能较好，因为它需要维护一个链表以保持Map中key-value对的添加顺序。

HashMap与Hashtable的区别

  • HashMap继承至AbstractMap，AbstractMap实现了Map接口；Hashtable继承至Dictionary，实现了Map接口。
  • HashMap不是线程安全的；Hashtable是线程安全的。
  • HashMap支持key和value为null；Hashtable不支持。
  • HashMap对key的hashcode进行二次hash，然后对数组长度取模；Hashtable直接用key的hashcode对数组长度取模。
  • HashMap快于Hashtable，Hashtable已经被淘汰了。
  • HashMap初始容量为16，扩容为2倍；Hashtable初始容量为11，扩容为2倍+1。

    HashSet、HashMap、Hashtable默认的负载极限是0.7
